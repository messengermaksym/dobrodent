## 🎯 Виявлені проблеми та способи їх вирішення

### 1. Неоптимізовані зображення (Виконано [x])
* **Проблема:** Зображення `img_9885.jpg`, `img_9814.jpg` та `logo_transparent.png` важили занадто багато та затримували завантаження сторінки (LCP = 6.5 с).
* **Вирішення:** 
  - [x] Обрізано фотографії до потрібного розміру.
  - [x] Конвертовано файли у формат `.webp`.
  - [x] Дано файлам зрозумілі SEO-назви (`dobrodent-main-banner.webp`, `dobrodent_clinic.webp`, `logo-dobrodent.webp`).
  - [x] Розмір файлів зменшено на ~80% (заощаджено >600 КБ).

### 2. Відсутність атрибутів `width` та `height` (Виконано [x])
* **Проблема:** Браузер вимагав явних розмірів для зображень для запобігання затримкам візуалізації.
* **Вирішення:** 
  - [x] Оновлено компонент `ImageWithPlaceholder.tsx` (додано підтримку `width` та `height`).
  - [x] Передано конкретні розміри для банера, блоку про клініку та логотипа у коді.

### 3. Попередження про кешування TTL (Інформативно)
* **Проблема:** GitHub Pages ставить короткий термін кешування файлів.
* **Вирішення:** Оскільки сайт розміщено на статичному хостингу GitHub Pages, змінити конфігурацію заголовків сервера неможливо. Це не критично після стиснення всіх зображень.
