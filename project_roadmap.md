# План розвитку проєкту ДоброДЕНТ 🚀

Нижче наведено структурований список задач, які ми плануємо реалізувати в майбутньому, а також декілька пропозицій для покращення.

## 📋 Основні заплановані задачі

- [ ] **Адмін-панель**
  Створення зручного інтерфейсу для самостійного редагування текстів, фотографій спеціалістів, галереї та прайс-листа.
  > [!TIP]
  > **Варіанти реалізації (коротко):**
  > 1. **TinaCMS / Decap CMS:** Ідеально для статики (зберігає зміни прямо в коді), безкоштовно.
  > 2. **Sanity / Strapi:** Професійна Headless CMS, дуже потужна, але вимагає стороннього хостингу/бази.
  > 3. **Google Sheets:** Найдешевший та найзрозуміліший варіант (особливо круто для прайсу).

- [x] **Попап форма (Модальне вікно)**
  Спливаюче вікно для швидкого запису на прийом (форма зв'язку), яке з'являтиметься поверх сторінки при натисканні на кнопки (наприклад, "Записатися"), не змушуючи користувача переходити на іншу сторінку.

- [ ] **Односторінкова версія (Landing Page)**
  Створення нової гілки (branch) в Git, де ми з'єднаємо всі поточні сторінки (Послуги, Прайс, Галерея, Команда, Контакти) в одну довгу безперервну сторінку з плавним скролінгом між секціями.

- [x] **Базова перевірка та налаштування SEO**
  Генерація `sitemap.xml`, `robots.txt`, додавання правильних мета-описів, та налаштування OpenGraph (щоб при пересиланні посилання в Telegram/Viber підтягувалося красиве лого та опис).

- [ ] **Підключення веб-аналітики**
  Інтеграція скриптів Google Analytics 4 (GA4) та/або Facebook Pixel для відстеження відвідуваності та "цільових дій" (хто натискав на номери телефонів, хто заповнив форму).

- [ ] **Інтеграція форми з Telegram-ботом**
  Надсилання заявок із контактної та попап-форми моментально в Telegram адміністратору клініки для миттєвої реакції на запит пацієнта.

- [x] **Блок живих відгуків з Google Maps**
  Виведення віджета з реальними оцінками 5⭐ з Google Карток, що підвищує рівень довіри нових пацієнтів.

- [x] **Мікро-анімації під час прокручування**
  Додавання плавних анімацій появи елементів (fade-in) при скролінгу за допомогою бібліотеки *Framer Motion* для більшої преміальності сайту.
